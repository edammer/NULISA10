# NULISA10 plasma proteomics pipeline: stepwise methods summary

## Overview

This workflow processes NULISA plasma proteomic quantification data from 465 samples representing 10 developmental disorder geneotype groups plus healthy sibling controls. Across the six scripts, the pipeline imports raw normalized protein quantities (NPQ), constructs sample metadata, performs pre-adjustment quality control, regresss out batch effects, runs groupwise differential expression and trait-correlation analyses, builds protein- and sample-level WGCNA networks for subtype discovery, trains genotype and epilepsy prediction models, and finally performs repeated 80/20 hold-out cross-validation for epilepsy prediction.

The scripts are intended to be run in the following order:

1. `1.Load+Initial_VarianceParition.R`
2. `2.bootstrapRegression.R`
3. `3.DEP_volcanoes_bootRegr.R`
4. `4.standard+transposedWGCNA(subtyping)-FaundezNulisa10-OLrem+ds4-fullPipeline.R`
5. `5.Genotype_epilepsy prediction+Enrichment_of_genotypes_into_subtypes-NULISA10.plasma.R`
6. `6.Genotype_epilepsy prediction_confusion matrix-NULISA10.plasma.R`

## Stepwise methods summary

### Script 1. Data import, cohort annotation, and initial variance partitioning

**File:** `1.Load+Initial_VarianceParition.R`

This script reads the starting CSV file, separates metadata from assay data, and establishes the working cohort structure. Metadata are extracted from the first 13 rows and transposed into a sample-by-variable data frame (`numericMeta`), while assay rows are converted to numeric values and stored as an assay-by-sample matrix (`exprMat0`). A categorical disease-group variable (`Group`) is created by assigning controls to `MutationStatus == "Control"` and replacing diseased labels with their causal gene names. The script then performs pre-adjustment quality control by variance partitioning using a mixed-effects model that includes age, sex, amyloid-β42, mutation group, experiment, foundation batch, and APOE4 carrier status. The resulting variance fractions are visualized as a global variance partition plot and as ranked bar plots highlighting the assays most strongly driven by group, foundation, experiment, APOE4, AB42, age, or sex. The script saves the full workspace as `1.FaundezNULISA.RData` for downstream reuse.

**Primary outputs:**
- `1.FaundezNULISA.RData`
- `1.VariancePartition-NULISA-130x465_unregr.pdf`

---

### Script 2. Bootstrap regression-based batch adjustment and post-adjustment QC

**File:** `2.bootstrapRegression.R`

This script loads `1.FaundezNULISA.RData` and applies assay-wise regression to remove technical batch structure while preserving biological disease-group differences. The unadjusted expression matrix (`exprMat0`) is copied to `cleanDat.unreg`. Batch covariates are encoded as binary indicators for `Experiment` and `Foundation`, and each assay is fit using an ordinary nonparametric bootstrap linear regression (`R = 1000`) implemented in parallel across assays. For each assay, the median bootstrap coefficient for each batch term is estimated, and the regressed expression matrix is reconstructed by subtracting the estimated experiment and foundation effects. Distributional sanity checks are performed by comparing sample quantiles before and after adjustment. The batch-adjusted matrix is assigned to `cleanDat`, after which variance partitioning is repeated with the same covariate set used in Script 1 to confirm attenuation of batch-driven variance and retention of biologically relevant signal. The updated workspace is saved as `2.FaundezNULISA_regr.RData`.

**Primary outputs:**
- `2.FaundezNULISA_regr.RData`
- `2.VariancePartition-NULISA-130x465_Regressed(Experiment+Foundation).pdf`

---

### Script 3. Groupwise differential protein abundance, trait correlations, volcano plotting, and enrichment preparation

**File:** `3.DEP_volcanoes_bootRegr.R`

This script loads the batch-adjusted workspace (`2.FaundezNULISA_regr.RData`) and performs both visual and statistical downstream analyses. First, multidimensional scaling (MDS) is used to inspect global sample separation among the 10 gene-defined disease groups and controls. Next, the script sources `parANOVA.dex.R` and executes pairwise disease-versus-control differential expression analyses for each of the 10 mutation groups. Each analysis is restricted to one disease group and controls, and the per-group outputs are combined into a single assay-level table containing unadjusted p values, Benjamini–Hochberg adjusted p values, and log2 effect sizes across all disease comparisons. These results are visualized with volcano plots and written to a summary CSV.

The script also performs trait-correlation analysis using bicorrelation against selected molecular and clinical traits, including HBA1, NEFL, oligomeric α-synuclein, APOE4 carrier status, and epilepsy status. A combined correlation table (`CORout`) is generated, capped for visualization stability at extremely small p values, and plotted with the same volcano framework. Finally, enrichment helpers from `GOparallel-FET.R` are sourced to support optional ontology analysis of correlation and network outputs, and the complete analysis state is saved as `3.FaundezNULISA-DEPandBicor.RData`.

**Primary outputs:**
- `3.MDS_sampleDifferencesInNULISAdata(... )_bootRegr.pdf`
- `3.FDR(BH-corrected_p_values)_and_log2EffectSizes-NULISA_130assays-10gene-drivenDiseaseSampleSets_vs_148ControlSamples.csv`
- volcano plot PDFs / HTMLs for differential and correlation analyses
- `3.FaundezNULISA-DEPandBicor.RData`

### Script 4. Protein-level WGCNA, iterative module cleanup, transposed sample-level WGCNA, and subtype definition

**File:** `4.standard+transposedWGCNA(subtyping)-FaundezNulisa10-OLrem+ds4-fullPipeline.R`

This script loads `3.FaundezNULISA-DEPandBicor.RData` and performs two related network analyses.

#### 4A. Protein-level coexpression network

The batch-adjusted assay-by-sample matrix is first filtered for sample outliers using a bicorrelation-based sample network. Connectivity z scores are iteratively recomputed, and samples falling more than 3 standard deviations below the mean connectivity are removed until no further outliers remain. Assays with at least 50% missing values are then excluded. Using the retained data matrix, the script selects a soft-thresholding power from the scale-free topology fit curve and constructs a signed bicorrelation WGCNA network with `blockwiseModules` (deepSplit = 4, minimum module size = 6, merge cut height = 0.07). Module eigengenes and signed kME values are calculated, followed by an iterative cleanup procedure that moves weak intramodular members to grey, reassigns grey members with strong module affinity, and reorders module colors by decreasing size. A global network PDF and a tab-delimited kME / module-membership table are written, and an intermediate saved image is produced.

#### 4B. Sample-level transposed network and subtype assignment

The script then returns to the full 465-sample matrix, transposes it so that samples act as network nodes, and runs a second signed bicorrelation WGCNA to identify sample subtypes. The same eigengene and kME cleanup framework is applied, yielding five sample-level subtypes (turquoise, blue, brown, yellow, green). Subtype membership is recorded in `numericMeta` as both a categorical variable (`sampleSub`) and one-hot encoded subtype indicators. Additional annotation colors are built for sex, APOE4 status, epilepsy, and mutation group, and subtype-aware global network plots are produced. The full subtype workspace is saved as `4b.saved.image.NULISA10.plasma-eigensampleWGCNA.Rdata`.

**Primary outputs:**
- `4.saved.image.NULISA10.plasma-eigenproteinWGCNA.Rdata`
- `4.GlobalNetworkPlots-...pdf`
- `4.ProteinAssay(130)-ModuleAssignments-...txt`
- `4b.saved.image.NULISA10.plasma-eigensampleWGCNA.Rdata`
- `4b.GlobalNetworkPlots-...pdf`
- sample subtype heatmaps / clustering plots

### Script 5. Mutation-group prediction, epilepsy prediction, subtype comparisons, and subtype-by-genotype enrichment

**File:** `5.Genotype_epilepsy prediction+Enrichment_of_genotypes_into_subtypes-NULISA10.plasma.R`

This script loads the subtype-aware workspace (`4b.saved.image.NULISA10.plasma-eigensampleWGCNA.Rdata`) and sources `binaryPredictor.R`.

#### 5A. Stage-1 feature ranking and stage-2 multiclass genotype prediction

For each mutation group plus controls, the script trains a one-vs-rest binary ensemble predictor using all available assays to derive ranked feature-importance tables (`rankedProteins`). These ranked assay lists are then used as prior feature sets for a second-stage multiclass classifier (`train_NULISA10_full`), which is trained on the top-ranked assays per class (for example, top 10 or top 5 features) with a target positive predictive value threshold of 0.80. Predictions are generated for all samples, enabling comparison between assigned mutation group and known group labels.

#### 5B. Epilepsy prediction

The script then subsets to samples with known epilepsy status and relabels positive cases as `Epilepsy`. A new set of stage-1 one-vs-rest binary predictors is trained for `Epilepsy` and `No`, generating ranked epilepsy-associated features. These feature rankings are used in a second-stage binary ensemble to predict epilepsy status from the top assays. Predictions are applied to samples with known and unknown epilepsy labels, and the predicted epilepsy status is merged back into `numericMeta`.

#### 5C. Differential comparison of KANSL1-enriched subtypes and enrichment of genotype labels within subtypes

The script also defines two manually curated KANSL1 subtype groups (`KANSL1.yellow` and `KANSL1.blue`) from the sample-level WGCNA output, performs differential protein abundance analysis between these two subtype subsets using the previously defined ANOVA/volcano framework, and overlays protein network colors onto the resulting statistics. To quantify genotype enrichment across the five discovered sample subtypes, the script writes sample-by-genotype and sample-by-subtype membership tables and uses `geneListFET.R` to perform Fisher exact tests for overlap enrichment between subtype assignments and known genotype categories.

**Primary outputs:**
- ranked genotype feature tables in `rankedProteins`
- multiclass group predictions on all samples
- epilepsy feature rankings and predicted epilepsy labels
- `samplesByGenotype_forFET.csv`
- `samplesBySubtype.csv`
- subtype-versus-genotype Fisher exact test heatmaps
- `saved.image02-03-2026.RData`

### Script 6. Repeated 80/20 hold-out cross-validation for epilepsy prediction with confusion-matrix output

**File:** `6.Genotype_epilepsy prediction_confusion matrix-NULISA10.plasma.R`

This script is a dedicated validation extension of Script 5. It loads the subtype-aware workspace (`4b.saved.image.NULISA10.plasma-eigensampleWGCNA.Rdata`) and sources `binaryPredictor.noRankedProteins.prior.R`. A hardened version of `train_NULISA10_full` is defined to tolerate feature mismatches across folds by resolving features against available columns, padding missing columns at prediction time, and forcing subsets eagerly before parallel evaluation.

The script then defines helper functions for epilepsy-specific feature ranking and implements `epilepsy_binary_holdout_cv`, a repeated 80/20 hold-out cross-validation framework. In the recommended mode, top epilepsy features are re-ranked inside each outer training split to avoid information leakage. In an alternate sensitivity-analysis mode, a fixed set of top epilepsy features derived from all labeled samples can be reused across all folds. For each outer split, a single binary `Epilepsy`-versus-`No` predictor is trained on the selected top features, applied to the 20% hold-out subset, and evaluated by a 2 × 2 confusion matrix. Per-fold and pooled performance summaries are returned, including TP, FP, FN, TN, accuracy, sensitivity, specificity, PPV, NPV, F1 score, and balanced accuracy, along with mean and standard deviation of the confusion-matrix cells across folds.

**Primary outputs:**
- repeated-hold-out epilepsy prediction summary object
- pooled 2 × 2 confusion matrix
- per-fold performance table
- fixed top epilepsy feature list for sensitivity analysis

## Minimal stepwise run guide

### Step 1
Run `1.Load+Initial_VarianceParition.R` using `FaundezNULISA_forR.csv` as the starting input.

### Step 2
Run `2.bootstrapRegression.R` after Step 1 completes. This script requires `1.FaundezNULISA.RData`.

### Step 3
Run `3.DEP_volcanoes_bootRegr.R` after Step 2 completes. This script requires `2.FaundezNULISA_regr.RData`, `parANOVA.dex.R`, and `GOparallel-FET.R`.

### Step 4
Run `4.standard+transposedWGCNA(subtyping)-FaundezNulisa10-OLrem+ds4-fullPipeline.R` after Step 3 completes. This script requires `3.FaundezNULISA-DEPandBicor.RData`.

### Step 5
Run `5.Genotype_epilepsy prediction+Enrichment_of_genotypes_into_subtypes-NULISA10.plasma.R` after Step 4 completes. This script requires `4b.saved.image.NULISA10.plasma-eigensampleWGCNA.Rdata`, `binaryPredictor.R`, and `geneListFET.R`.

### Step 6
Run `6.Genotype_epilepsy prediction_confusion matrix-NULISA10.plasma.R` after Step 4 or after Step 5, depending on whether only epilepsy hold-out validation is needed. This script requires `4b.saved.image.NULISA10.plasma-eigensampleWGCNA.Rdata` and `binaryPredictor.noRankedProteins.prior.R`.

## Additional Notes

- The scripts operate on a plasma NULISA NPQ matrix with 130 assays and 465 samples after import.
- Mutation-group comparisons are structured against healthy sibling controls.
- Batch correction is performed before WGCNA, differential testing, and machine learning.
- Protein-level WGCNA and transposed sample-level WGCNA are both used: the former for assay modules and the latter for sample subtype discovery.
- Script 5 is primarily a discovery / deployment script, whereas Script 6 provides the more rigorous out-of-sample validation framework for epilepsy prediction.
